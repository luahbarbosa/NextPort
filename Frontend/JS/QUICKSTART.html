Step 1: Visualizar o Dashboard

Abra em um navegador

Simplesmente abra o arquivo index.html em seu navegador favorito:

file:///caminho/para/Frontend/JS/index.html

Nota: Para melhor performance e recursos (como fetch), use um servidor local.

Ou use um servidor local

Recomendamos usar um servidor de desenvolvimento:

# Option 1: Node.js (http-server) npx http-server Frontend/JS --port 8080 # Option 2: Python 3 python -m http.server 8000 --directory Frontend/JS # Option 3: Python 2 python -m SimpleHTTPServer 8000

Depois acesse: http://localhost:8080 (ou 8000)

Step 3: Integrar com Backend

3.1 - Preparar Arquivo de Configuração

Copie o arquivo de exemplo:

cp Frontend/JS/api-config.example Frontend/JS/api-config.js

Edite api-config.js com sua URL de API:

const API_CONFIG = { BASE_URL: 'https://sua-api.com/v1', TIMEOUT: 10000 };

3.2 - Descomente as Chamadas de API

Em dashboard.js, localize as funções de fetch e descomente as chamadas reais:

// Em dashboard.js, função fetchDashboardData() // Descomente: const response = await fetch('https://sua-api.com/v1/dashboard'); const data = await response.json(); // Comente: // return mockData;

3.3 - Endpoints Esperados

Certifique-se de que seu backend implementou estes endpoints:

GET /api/v1/dashboard # Dados do dashboard GET /api/v1/network-metrics # Métricas de rede GET /api/v1/devices # Listar dispositivos POST /api/v1/devices # Criar dispositivo GET /api/v1/devices?status=online # Filtrar

❓ FAQ (Dúvidas Frequentes)

D: Como mudar as cores do dashboard?

R: Edite as variáveis CSS em styles.css seção :root

D: Como adicionar um novo item de menu?

R: Veja exemplos em customization-guide.js função addMenuItemDynamically

D: Qual navegador é suportado?

R: Todos os navegadores modernos (Chrome, Firefox, Safari, Edge). IE11 não é suportado.

D: Como testar sem o backend?

R: Por padrão, usa mock data. Mude VITE_USE_MOCK_DATA=false em .env quando tiver API.

D: Como fazer deploy?

R: Copie os arquivos para um servidor web ou use serviço como Netlify, Vercel.

🆘 Encontrou um Erro?

Abra o Console do Navegador (F12)

O código inclui logs detalhados. Verifique:
1. Network tab - Requisições de API
2. Console tab - Erros JavaScript
3. Application tab - LocalStorage/Cache

Passos para Debug

  1. Abra F12 → Console
  2. Procure por mensagens de erro (❌)
  3. Verifique Network tab para requisições falhadas
  4. Leia README.md seção "Debugging & Logging"
  5. Verifique se .env está configurado corretamente
